Installation and Setup

Getting Ruby on Your System

Before we can start programming in Ruby, we need to install it on our system. Ruby is available for all major operating systems.

On macOS:

macOS comes with Ruby pre-installed, but it's usually an older version. To install the latest version, you can use Homebrew:

$ brew install ruby

On Windows:

Download and install Ruby using RubyInstaller from rubyinstaller.org

On Linux:

$ sudo apt-get install ruby-full  # For Debian/Ubuntu
$ sudo yum install ruby  # For CentOS/RHEL

Verifying Your Installation

Once installed, you can verify your Ruby version by opening a terminal or command prompt and typing:

$ ruby -v

Interactive Ruby (IRB)

Ruby comes with an interactive shell called IRB. This is a great place to test small pieces of code and experiment.

$ irb
irb(main):001:0> puts "Hello, Ruby!"
Hello, Ruby!
=> nil

Creating Your First Ruby Program

Let's create a simple "Hello World" program:

  1. Create a file named hello.rb
  2. Add this code:
#!/usr/bin/env ruby

puts "Hello, Ruby World!"

Run it with:

$ ruby hello.rb

Variables and Data Types

Variables in Ruby

Variables in Ruby are used to store data. Unlike many other languages, Ruby variables don't need to be declared with a specific type.

name = "Ruby"
age = 30
is_programming_language = true

Naming Conventions

Ruby variable names follow these conventions:

  • They are case sensitive (name and Name are different variables)
  • They can contain letters, numbers, and underscores
  • They should start with a lowercase letter or underscore
  • By convention, multi-word variables use snake_case: user_name, first_name

Core Data Types

Numbers

# Integers
age = 30
temperature = -5

# Floating point numbers
pi = 3.14159
weight = 65.5

Strings

# String literals
name = "Ruby"
message = 'Hello, World!'

# String interpolation (only works with double quotes)
greeting = "Hello, #{name}!"  # => "Hello, Ruby!"

Booleans

is_valid = true
has_errors = false

Symbols

Symbols are lightweight strings, often used as keys in hashes.

status = :success
direction = :north

Arrays

fruits = ["apple", "banana", "cherry"]
mixed_array = [1, "two", :three, 4.0]

# Accessing elements (zero-indexed)
fruits[0]  # => "apple"

Hashes

person = {
  "name" => "Alice",
  "age" => 30,
  "occupation" => "Developer"
}

# Modern syntax with symbols as keys
person = {
  name: "Alice",
  age: 30,
  occupation: "Developer"
}

# Accessing values
person[:name]  # => "Alice"

nil

nil represents the absence of a value in Ruby, similar to null in other languages.

empty_value = nil

Type Conversion

# String to Integer
"123".to_i  # => 123

# Integer to String
42.to_s  # => "42"

# String to Float
"3.14".to_f  # => 3.14

Checking Data Types

# Using .class method
42.class  # => Integer
"hello".class  # => String

# Using .is_a? method
42.is_a?(Integer)  # => true
"hello".is_a?(String)  # => true

Basic Operators

Arithmetic Operators

Ruby provides standard arithmetic operators for mathematical operations:

a = 10
b = 3

addition = a + b  # => 13
subtraction = a - b  # => 7
multiplication = a * b  # => 30
division = a / b  # => 3 (integer division)
division_float = a / b.to_f  # => 3.3333333333333335 (floating point division)
modulus = a % b  # => 1 (remainder of division)
exponentiation = a ** b  # => 1000 (a raised to the power of b)

Assignment Operators

x = 5  # Basic assignment

# Compound assignment
x += 3  # Same as x = x + 3, now x is 8
x -= 2  # Same as x = x - 2, now x is 6
x *= 4  # Same as x = x * 4, now x is 24
x /= 3  # Same as x = x / 3, now x is 8
x %= 5  # Same as x = x % 5, now x is 3
x **= 2  # Same as x = x ** 2, now x is 9

Comparison Operators

a = 10
b = 5
c = 10

a == c  # => true (equality)
a != b  # => true (inequality)
a > b   # => true (greater than)
a < b   # => false (less than)
a >= c  # => true (greater than or equal to)
a <= b  # => false (less than or equal to)

# Spaceship operator (returns -1, 0, or 1)
a <=> b  # => 1 (a > b)
a <=> c  # => 0 (a == c)
b <=> a  # => -1 (b < a)

Logical Operators

true && true    # => true (AND: both operands are true)
true && false   # => false

true || false   # => true (OR: at least one operand is true)
false || false  # => false

!true   # => false (NOT: negation)
!false  # => true

# Alternative syntax
true and true   # => true
true or false   # => true
not true        # => false

Note: and, or, and not have lower precedence than &&, ||, and !.

String Operators

first_name = "Ruby"
last_name = "Programming"

# String concatenation
full_name = first_name + " " + last_name  # => "Ruby Programming"

# String multiplication
separator = "-" * 10  # => "----------"

# String interpolation
message = "#{first_name} is fun!"  # => "Ruby is fun!"

Range Operators

# Inclusive range (includes both start and end values)
(1..5).to_a  # => [1, 2, 3, 4, 5]

# Exclusive range (excludes the end value)
(1...5).to_a  # => [1, 2, 3, 4]

# Character ranges
('a'..'e').to_a  # => ["a", "b", "c", "d", "e"]

Mini Project: Temperature Converter

Create a Ruby script that:

  1. Defines a temperature in Celsius
  2. Converts it to Fahrenheit using the formula: F = C * 9/5 + 32
  3. Displays both temperatures
  4. Compares the two temperatures and displays whether the Fahrenheit value is greater than, less than, or equal to the Celsius value

Bonus: Make the script convert Fahrenheit to Celsius as well!

Solution:
# Chapter 1: Mini Project - Temperature Converter

# Define a temperature in Celsius
celsius_temp = 30

# Convert it to Fahrenheit using the formula: F = C * 9/5 + 32
fahrenheit_temp = (celsius_temp * 9.0 / 5.0) + 32.0

# Display both temperatures
puts "Temperature in Celsius: #{celsius_temp}°C"
puts "Temperature in Fahrenheit: #{fahrenheit_temp}°F"

# Compare the two temperatures and display whether the Fahrenheit value is
# greater than, less than, or equal to the Celsius value
if fahrenheit_temp > celsius_temp
  puts "The Fahrenheit temperature (#{fahrenheit_temp}°F) is greater than the Celsius temperature (#{celsius_temp}°C)."
elsif fahrenheit_temp < celsius_temp
  puts "The Fahrenheit temperature (#{fahrenheit_temp}°F) is less than the Celsius temperature (#{celsius_temp}°C)."
else
  puts "The Fahrenheit temperature (#{fahrenheit_temp}°F) is equal to the Celsius temperature (#{celsius_temp}°C)."
end

# BONUS: Convert Fahrenheit to Celsius as well
puts "\n--- BONUS: Fahrenheit to Celsius ---"
original_fahrenheit = 98.6
original_celsius = (original_fahrenheit - 32.0) * 5.0 / 9.0
puts "Temperature in Fahrenheit: #{original_fahrenheit}°F"
puts "Converted to Celsius: #{original_celsius}°C"

Input/Output Operations

Output Methods

Ruby provides several methods for displaying output:

puts

The puts method prints the given object(s) followed by a newline character.

puts "Hello, Ruby!"
# Output: Hello, Ruby!
# (with a newline at the end)

# Multiple arguments
puts "First line", "Second line"
# Output:
# First line
# Second line

print

The print method displays output without adding a newline at the end.

print "Hello, "
print "Ruby!"
# Output: Hello, Ruby! (on the same line)

p

The p method displays the raw representation of an object, which is useful for debugging.

puts "Hello\nWorld"
# Output:
# Hello
# World

p "Hello\nWorld"
# Output: "Hello\nWorld" (shows the actual string with escape characters)

printf

For formatted output, similar to C's printf:

name = "Ruby"
age = 30
printf("Name: %s, Age: %d\n", name, age)
# Output: Name: Ruby, Age: 30

Input Methods

gets

The gets method reads a line of input from the user, including the newline character at the end.

print "Enter your name: "
name = gets
puts "Hello, #{name}"
# If user enters "Alice", output will be:
# Hello, Alice
# (with an extra newline)

gets.chomp

To remove the trailing newline, we use chomp:

print "Enter your name: "
name = gets.chomp
puts "Hello, #{name}!"
# If user enters "Alice", output will be:
# Hello, Alice!

Converting Input

Input from gets is always a string, so we need to convert it for numeric operations:

print "Enter your age: "
age = gets.chomp.to_i  # Convert to integer
puts "In 5 years, you'll be #{age + 5} years old."

Working with Files

Reading from a File

# Reading entire file at once
content = File.read("example.txt")
puts content

# Reading line by line
File.foreach("example.txt") do |line|
  puts line
end

# Using File.open with a block
File.open("example.txt", "r") do |file|
  while line = file.gets
    puts line
  end
end  # File is automatically closed when the block ends

Writing to a File

# Writing to a file (overwrites existing content)
File.write("output.txt", "Hello, Ruby!")

# Appending to a file
File.open("output.txt", "a") do |file|
  file.puts "This is a new line"
end

# Creating a new file with multiple lines
File.open("new_file.txt", "w") do |file|
  file.puts "Line 1"
  file.puts "Line 2"
  file.puts "Line 3"
end

Mini Project: Interactive Journal

Create a Ruby script that:

  1. Asks the user for their name
  2. Displays a welcome message
  3. Asks the user to enter a journal entry for today
  4. Saves the entry to a file named with today's date (e.g., "2025-05-27-journal.txt")
  5. Displays a confirmation message with the number of characters in their entry

Bonus: Add an option to read previous journal entries by entering a date.

Solution:
# Chapter 1: Mini Project - Interactive Journal

# Ask the user for their name
puts "Welcome to your Interactive Journal!"
print "Please enter your name: "
user_name = gets.chomp

# Display a welcome message
puts "Hello, #{user_name}! Let's write in your journal today."

# Get today's date
require 'date'
today = Date.today
date_string = today.strftime("%Y-%m-%d")

# Ask user for journal entry
puts "Please enter your journal entry for today (#{date_string}):"
puts "(Press Enter twice when finished)"

# Collect the entry (continue until user enters a blank line)
entry_lines = []
while true
  line = gets.chomp
  break if line.empty?
  entry_lines << line
end

journal_entry = entry_lines.join("\n")

# Save entry to a file
filename = "#{date_string}-journal.txt"
File.open(filename, "w") do |file|
  file.puts "Date: #{date_string}"
  file.puts "Name: #{user_name}"
  file.puts "Entry:"
  file.puts journal_entry
end

# Display confirmation message
puts "Journal entry saved to #{filename} (#{journal_entry.length} characters)"
